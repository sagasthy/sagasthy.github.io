Safety Framework: Risk Levels

The agent classifies every issue and enforces appropriate gates:

Risk Level Examples Action Compliance
AUTO_FIX Typos, syntax errors, linting, version bumps Auto-create PR, run tests, optional auto-merge Logged for audit
NEEDS_REVIEW DB migrations, config changes, test updates Create PR, flag for human approval Mandatory review
MANUAL_ONLY Payment logic, blockchain, security code Create GitHub Issue, no PR Zero automation
Broadridge Compliance: All AI-generated changes logged with decision rationale. Payment/blockchain/compliance code never auto-fixed. Code review required before merge.

Risks & Mitigations

Risk Impact Mitigation
Agent proposes wrong fix Code quality issues in PR Linting + type checking before PR. Human review required for NEEDS_REVIEW level.
False confidence on risky code Compliance violation Conservative risk policies. Payment/compliance logic ‚Üí MANUAL_ONLY. Audit logging.
Webhook overload Pod resource exhaustion Horizontal scaling + queue fallback. Rate limiting on webhook receiver.
Anthropic API quota spike Cost overrun Monthly budget tracking. Defer non-urgent analyses. Batch processing for off-peak.
False negatives (missed fixes) Agent fails to help Graceful degradation. Issues created for manual review. No harm.
Philosophy: Conservative by default. Better to over-flag for human review than auto-fix something risky.
